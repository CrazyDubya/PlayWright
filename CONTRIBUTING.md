# Contributing to PlayWright

Thank you for your interest in contributing to PlayWright! We welcome contributions that improve the framework, tools, and documentation.

If you want to share musicals you've created using the PlayWright framework:
- **You retain full copyright** to your own original works
- You may share them in your own repositories
- We may consider featuring your work (with your permission) as examples
- Any shared work would need a clear licensing statement from you

## ü§ù How to Contribute

### Types of Contributions We Welcome

1. **Framework Improvements**
   - Bug fixes in tools and scripts
   - New features for the framework
   - Performance improvements
   - Code quality enhancements

2. **Documentation**
   - Tutorial improvements
   - Clarifications and corrections
   - New guides and examples
   - Translation of documentation

3. **Templates and Tools**
   - New templates for musical creation
   - Improved automation scripts
   - Better workflows and processes
   - Quality assurance tools

4. **Community Support**
   - Answering questions in discussions
   - Helping other users
   - Sharing best practices (without copyrighted content)

### Contribution Process

1. **Fork the Repository**
   - Create your own fork of the code

2. **Create a Branch**
   - Use a descriptive branch name (e.g., `fix-validator-bug`, `add-song-template`)

3. **Make Your Changes**
   - Follow existing code style and conventions
   - Test your changes thoroughly
   - Update documentation as needed

4. **Submit a Pull Request**
   - Clearly describe your changes
   - Reference any related issues
   - Ensure all tests pass (if applicable)

5. **License Agreement**
   - By submitting a PR, you agree that your contributions will be licensed under CC BY-NC-SA 4.0
   - You confirm that you have the right to submit the code under this license
   - You understand that you cannot revoke this license for your contributions

## ‚ùå What We Don't Accept

- Modifications to the copyrighted musical examples in `/projects/` and `/examples/`
- Contributions that violate the licensing terms
- Code that includes copyrighted material without proper rights
- Contributions that would require changing to a more restrictive license

## üé≠ Sharing Your Own Musicals

If you've created amazing musicals using PlayWright:

1. **Keep them in your own repository** - You own them!
2. **License them however you want** - They're yours
3. **Let us know** - We'd love to see what you've created (and may feature it with your permission)
4. **Consider sharing insights** - You can contribute learnings back to the framework without sharing the actual musical content
